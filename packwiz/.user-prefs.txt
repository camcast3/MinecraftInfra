# packwiz/.user-prefs.txt
#
# Curated list of pack-shipped files that players typically tune (mod
# graphics, shaders, recipe-viewer UI, map style, HUD position, etc.).
#
# Consumed by infra/azure/scripts/publish-prism-pack.ps1: read this file,
# write the entries as JSON into the published Prism zip at
#   <InstanceName>/.negativezone/preserve-list.json
# Consumed by docs/assets/update.ps1: after the atomic .minecraft swap,
# restore any of these paths from .minecraft.bak/ so user tweaks survive
# pack updates. Default Options mod handles vanilla files (options.txt,
# keybindings) separately via config/defaultoptions/ — those do NOT
# belong in this list. servers.dat is ALSO excluded but for a different
# reason: it's a single-server pack and the build pipeline writes it
# directly (infra/azure/scripts/build-instance-from-packwiz.ps1 calls
# Set-NegativeZoneServersDat after packwiz install), so pack-author
# updates to the server list — name change, DNS migration, additional
# Velocity backend — propagate to existing players on the next update.
# Consumed by docs/assets/backup.ps1: widen snapshot scope so accidentally
# deleted preserve files are recoverable from a snapshot.
#
# Format: one path per line, relative to .minecraft/.
# - Blank lines ignored.
# - Lines starting with `#` are comments.
# - Whitespace trimmed.
#
# Curation rule of thumb: a file belongs here ONLY if all three are true:
#   1. The pack ships a sensible default (not auto-generated by the mod).
#   2. Players commonly tune it (graphics tier, shader choice, map style,
#      HUD layout, sound prefs, animation toggles, etc.).
#   3. Pack-author balance updates to the file would NOT need to be pushed
#      to existing players (i.e., we accept "first install seeds, never
#      overwritten" semantics for this file).
# When in doubt, leave it OUT. False-negatives recover with one tweak;
# false-positives silently break balance pushes for existing players.
